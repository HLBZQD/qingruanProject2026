## 当前审查结果

本轮组件B诊断报告（b_v3_diag_v2.md）经质询确认（LOCATED），共发现8个质量问题：

1. **§11.2 依赖表中 env.d.ts 描述与 §2.2/§12 结论矛盾** — §11.2 第 673 行仍描述 env.d.ts 为待实现功能，但 §2.2 明确结论当前状态无问题、§12 已将 A1 标记为取消（0d），两处信息矛盾。严重程度：中。改进建议：从 §11.2 依赖表中删除 env.d.ts 行，或将其标注为"已取消（Vite 已内置 .vue 类型声明）"。

2. **§5.2 声称"JWT 相关逻辑整合在 authStore.ts"与代码事实不符** — authStore 仅将 token 作为字符串存储/读取/清除，不存在任何 JWT 解析逻辑（无 atob、base64 解码、payload 提取、exp 字段检查）。严重程度：中。改进建议：修正 §5.2 描述为"JWT 解析功能完全缺失——authStore 仅将 token 作为不透明字符串管理"，并同步重新评估 A4（useAuth composable）的工作量估算（0.3d 可能偏低）。

3. **A8 修复方案描述不精确，无法按描述实现设计意图** — §11.2/§12 描述最简方案为"修改 next('/home') 为 next(false) 语义，即：无来源页时回 /home，有来源页时停留在来源页"。但 Vue Router 的 next(false) 仅中止导航，不会在无来源页时回 /home，需额外逻辑。严重程度：低。改进建议：明确 A8 的代码变更内容，建议使用 `next(from && from.path !== to.path ? false : '/home')` 或等效逻辑。

4. **C1 文章收藏任务未分析状态管理层需求，可能低估实现范围** — 文章收藏涉及乐观更新、跨页面状态同步、收藏列表分页缓存等状态管理需求，但当前描述未覆盖实现位置的架构决策。严重程度：中。改进建议：补充 C1 任务的状态管理架构分析，明确收藏状态的管理层级（建议扩展现有 useArticleApi 为带状态管理的 composable 或新建 articleStore），基于选定架构重新确认 1.5d 估算的充分性。

5. **搜索方案（C2）未讨论缓存数据时效性问题** — §11.3 推荐全量拉取 GET /api/articles（pageSize=100）后在本地过滤，但未评估 homeStore 对文章数据实施的 sessionStorage 缓存（TTL 1h）对搜索结果时效性的影响——新发布文章在缓存过期前不会出现在搜索结果中。严重程度：低。改进建议：补充缓存时效性说明，建议搜索时始终请求 API 而非使用缓存。

6. **§2.1 和 §2.4 的基础设施偏离未纳入 §13 架构偏离汇总表** — §2.1 的 setupAxiosInterceptors() 未被显式调用偏离和 §2.4 的样式文件路径偏离在 §13 架构偏离汇总表中缺失。严重程度：低。改进建议：将上两项补充到 §13 架构偏离汇总表中，并对每项给出"维持现状并更新设计文档"或"对齐设计"的明确建议。

7. **§12 Phase 2 人力分配中 D3 的延迟启动未计入关键路径计算** — D 组依赖链规定 D3 建议在 D1 合入后启动，若 D1 延期，D3 将阻塞人 2 进度。严重程度：低。改进建议：为 D3 增加缓冲说明或显式将 D1 和 D3 分配给同一人。

8. **§14 优先级清单中"搜索功能"排序与该功能用户可感知程度不匹配** — Home.vue 首页首屏已渲染搜索栏，用户点击后收到"搜索功能开发中" Toast，符合 P0 定义"用户点击相关入口时无法完成预期操作"。严重程度：一般。改进建议：将搜索功能提升为 P1（理由：用户可感知的功能缺口），或保持 P2 但补充说明。
