## 当前审查结果

本轮组件B诊断报告识别出 6 个质量问题（全部经质询确认，质询结论：LOCATED）：

### 问题 1（严重程度：高）
**§11.2 行661 依赖描述与 §5.2 结论存在事实矛盾**

§11.2 依赖表 B1（用户注册）行中声明"JWT 解析已由 authStore 完成"，但 §5.2 明确结论为"代码中不存在任何 JWT 解析逻辑：无 atob 解码、无 base64 解码、无 payload 提取、无 exp 字段检查"。两处描述直接矛盾。

改进建议：将行661的"JWT 解析已由 authStore 完成"修改为"注册操作仅需存储后端返回的 token/role/user，不涉及 JWT 解析"——这与 §5.2 的事实一致，且该结论（B1 不依赖 A4）本身是正确的，仅表述有误。

### 问题 2（严重程度：高）
**§12 人力与工期估算存在多处数据不一致**

1. 主分配表显示 3 人方案中人2 累计 2.3d 为最长路径，人3（含C2）累计 2.0d；但行861 声称"C2（1.0d）无法在 3 人内消化"，与分配表中 C2 已分配给 人3 的事实矛盾。
2. 行864 汇总"3 人方案 Phase 2 ≥2.5d"，但主分配表最长路径仅为 2.3d。2.5d 仅出现在调度说明的替代分配推演中，与主分配表的结论不匹配。
3. 行859 称人2 的 2.3d"超过 3 人内最短工期"暗示需调整，但 Phase 持续时间由最长路径决定——人2 的 2.3d 本身就是 Phase 2 的工期，不存在"超过"关系。此表述混淆了单人负荷与阶段工期两个概念。

改进建议：明确选定一个推荐分配方案，基于该方案统一计算并汇总 Phase 工期。以主分配表为例，Phase 2 = 2.3d（max(1.5, 2.3, 2.0)），3 人总工期 = 0.5 + 2.3 + 0.1 = 2.9d。删除行861中与分配表矛盾的"C2 无法在 3 人内消化"表述。将行859的"超过 3 人内最短工期"改为客观描述。若需保留替代分配方案，应独立标注为"备选方案"并给出对应工期。

### 问题 3（严重程度：中）
**D3 迁移任务未分析 Admin.vue 内联 SSE 与 chatStore.sendAdminMessage 的行为等价性**

D3 任务要求 Admin.vue 废弃内联 SSE 解析逻辑并切换至 chatStore.sendAdminMessage，但未验证两者在事件处理上的行为等价性。两者的 SSE 事件分发逻辑可能对 admin 特有的 SSEAgentMessageEvent/SSEAgentThoughtEvent 等事件类型存在处理差异。

改进建议：在 D3 任务说明中补充行为等价性验证要求：①对比 Admin.vue 与 chatStore 的 SSE 事件分发分支，确认 admin 特有事件类型的处理路径一致；②迁移后至少执行一次 Admin 对话端到端冒烟测试。

### 问题 4（严重程度：中）
**AI 助手导航（navigate）的 P1 优先级依据不足**

§14 将"AI 助手导航（navigate）"列为 P1，但 chatStore.ts 的 navigate() 为空函数体，仅在 AI 回复含跨模块导航指令时触发。报告未评估：①当前 AI 工作流是否已集成并实际发送导航指令；②若无导航指令，此功能是否在任何用户交互中可被感知。

改进建议：验证 AI 工作流的导航指令集成状态。若 AI 已集成且实际发送导航指令，保持 P1；若 AI 尚未集成导航指令体系，降级为 P2 并备注"待 AI 集成导航指令后升级为 P1"。

### 问题 5（严重程度：低）
**§14 P0 项存在语义重叠——Login.vue 死链接与用户注册实为同一问题的两个方面**

§14 将"用户注册"（P0）和"Login.vue 死链接「立即注册」"（P0）列为两个独立 P0 项。后者是前者的症状表现——注册表单缺失直接导致"立即注册"链接为自环无反馈链接。将症状和根因列为两个 P0 项可能造成任务分解的混淆。

改进建议：将行902和行903合并为一条 P0 项，描述为"用户注册功能缺失（含 Login.vue 死链接）"，并在说明中标注：若注册功能开发周期较长，可在 B1 中先以临时 Toast 修补死链接问题，注册功能上线后整体替换。

### 问题 6（严重程度：低）
**导航守卫免责声明拒绝行为修复（A8）的最佳路径与最简路径选择缺乏指引**

A8 任务同时描述了两条路径：最简方案（仅修改 router/index.ts:137 的 next('/home') 为来源页保留逻辑）和最佳实践（等 A5 完成后将 disclaimer 函数移入 useUI）。但报告未给出"选哪条"的明确推荐。

改进建议：给出明确推荐——鉴于 A8 仅 0.1d（最简方案）且与 A5 解耦，建议优先执行最简方案（在 Phase 1 中完成），并在 §13 架构偏离表中标记"免责声明函数位置"偏离项的修复状态为"A8 最简方案已修复行为，架构迁移待 A5 完成后执行"。
