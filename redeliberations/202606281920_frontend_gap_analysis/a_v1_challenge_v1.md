## 质询要点（CHALLENGED 时存在）

- **问题**：第9节 API 覆盖率统计数据错误——声称26已覆盖/5未覆盖/31总计/83.9%，实际约25已覆盖/7未覆盖/32可操作端点/~78%
- **原因**：覆盖率是判断前端API层工作进度的关键指标。错误的统计数据（高估覆盖率约6个百分点，漏计2个未覆盖端点）可能导致并行开发分组中API层工作量被低估，进而影响整体工期规划
- **建议方向**：对第9节API覆盖率表格重新逐行核对计数，修正覆盖/未覆盖/总计数值和百分比；同时检查是否存在其他端点被遗漏统计（如设计文档3.1节是否定义了第9节表格未列出的端点）

- **问题**：路由模块（router/index.ts）缺少独立的模块差距分析
- **原因**：路由是Vue应用的骨架，任务描述将其列为与views/stores/composables并列的参考材料。缺少路由模块的系统分析意味着无法确认路由表、导航守卫、meta配置是否与设计文档完全对齐，也无法评估路由层是否存在额外需要修复的差距
- **建议方向**：对照设计文档1.6节路由表逐条核对实际 router/index.ts 的路径、组件映射、meta字段；验证全局导航守卫的5个校验步骤是否完整实现；检查懒加载语法、路由命名和嵌套路由配置的一致性
