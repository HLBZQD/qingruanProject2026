# 任务描述

## 任务
对 `reviews/202606261037_server_backend_review/todo.md` 中列出的 18 个后端代码问题（9 严重 + 9 一般）进行深入的问题定位诊断分析。

## 诊断依据
1. **首要依据**: `docs/2_detailed_design_v3.md` — 详细设计文档，包含完整的后端 API 规范、JWT 认证约定、错误处理约定、分页约定、数据模型、中间件规范
2. **源代码**: `server/` 目录下所有 JS 文件
3. **前序审议记录**: `implements/` 目录下 7 个批次（batch1-batch7）的审议式实现产物

## 诊断目标
对每个问题逐项分析：
- 问题根因（代码与设计文档的具体偏差在哪里）
- 影响范围（哪些功能/端点在什么条件下会受此问题影响）
- 严重性验证（该问题的严重程度是否评定准确，与设计文档的偏差程度如何）
- 与其他问题的关联性（是否存在因果链或同源问题）

## 输出要求
诊断报告应定位到根因，让修复者知道"改哪里"和"为什么"，但不涉及具体修复代码。
